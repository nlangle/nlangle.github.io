No more MAMP

Recently, I elected to ditch MAMP to use OS X built in local development environment. MAMP was running on Mavericks, but when I switched to Yosemite, things were never quite right. After putting this off for several months, I started working on a few projects that were just not working right. Rather than mess with MAMP any longer, I decided to start from scratch.

However, you can't just uninstall MAMP and expect this to work...

Ripping it out

  1. follow these set up instuctions for the OS X local development environment
  2. turn off the MAMP apache server
  3. copy or move any project files
    • FROM: /Applications/MAMP/htdocs
    • TO: your new directory (/Users/nlangle/Sites)
  4. Check what version of PHP Apache wants to use.
    In terminal run: 
    • which php
    • php --ini
    If either of these are pointed to MAMP, the PATH should be edited
  5. Edit the PATH (if needed)
    echo $PATH
    Mine looked like this: 
    export PATH=/Applications/MAMP/bin/php/php5.5.23/bin:/Applications/MAMP/bin/php/php5.5.23/bin:/usr/local/bin:/usr/bin:/bin:/usr/sbin:/sbin source /Users/nlangle/.bash_profile 
export PATH=/Applications/MAMP/bin/php/php5.5.26/bin:/Applications/MAMP/bin/php/php5.5.23/bin:/Applications/MAMP/bin/php/php5.5.23/bin:/usr/local/bin:/usr/bin:/bin:/usr/sbin:/sbin source /Users/nlangle/.bash_profile
    Update it by editng 
    ~/.bash_profile
    I changed the path to this: 
    export PATH=/usr/bin:/bin:/usr/sbin:/sbin
    After saving changes, run this: 
    source ~/.bash_profile
  6. Restart Apache 
    sudo apachectl restart
  7. Don't look back!
