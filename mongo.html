Installing Mongo with MAMP

A brief rundown on installing and using mongodb with MAMP on osx yosemite

this wasn't exactly straightforward, it took several steps from several online resources to get this going, here is the roadmap for my process

this process will have to be repeated on all major osx updates (mavericks -> yosemite) or anytime that mamp is updated

  1. install mongodb

    make sure mongo is installed. this was already done, but adding for sake of completeness

    use brew to do this

    brew install mongodb

  2. install autoconf & xcode

    these were also previously installed, again just adding for completeness

    brew install autoconf

    NOTE: xcode updates may require reinstall of command line tools

    xcode-select --install
    How to Install Command Line Tools in OS X Mavericks & Yosemite (Without Xcode)

  3. Download and Install PHP Source

    1. get php version 
      view MAMP webstart page and select php or view under MAMP preferences php tab
    2. find source from:
      http://www.php.net/releases/
    3. unzip downloaded package
    4. move contents to MAMP directory
      • this will likely require a new directory to be created
      • cd /Applications/MAMP/bin/php/php5.5.26/
        mkdir include
        cd include
        mkdir php
      • move contents from downloaded php5.5.26 (or your php version) to this file. OR... simply move and rename the downloaded file to the "include" directory and rename it "php"
    5. open the php file and run ./configure
      6. cd /Applications/MAMP/bin/php/php5.5.26/include/php/ && ./configure
    6. the above step may encounter an -iconv error. if so run this:
      7. cd /Applications/MAMP/bin/php/php5.5.26/include/php/ && ./configure --without-iconv

  4. Build & Install mongo-php Driver

    install mongo

    cd /Applications/MAMP/bin/php/php5.5.26/bin
    sudo pecl install mongo

    wait for it...

  5. Add mongo.so extension to php.ini

    MAMP has LOTS of .ini files

    find the .ini files that match your version of php

    cd /Applications/MAMP/bin/php/php5.5.26/conf/php.ini

    find the "Extensions" section

    add this line to the bottom of the section

    extension=mongo.so

    but... mamp has another .ini file for this php version. for good measure, let's add the extension here too:

    cd /Applications/MAMP/conf/php5.5.26/php.ini

    same as above, add to the bottom of the extensions section

    extension=mongo.so

  6. Restart MAMP

    after restart verify mongo is installed:

    visit the mamp php page, look for mongo section on page
